Key Design Decisions:

  • Android Frontend: Kotlin + Jetpack Compose for modern declarative UI
  • Firebase Backend: Serverless architecture - no custom server needed
  • AI via Gemini: Google's Gemini 2.5 Flash for CV parsing, skill matching, interview scoring, MCQ generation
  • Python NLP: Standalone module for offline/alternative processing using sentence embeddings
  • Real-time: Firestore listeners for live updates on applications, notifications

Key Backend Flows

Job Application Flow

1. Student browses jobs → JobRepository.getAllActiveJobs()
2. Student taps Apply → FileUploadService.uploadResume() extracts text
3. CVParsingService.parseAndMatchCV() → Gemini AI parses PDF → CVParseResult
4. Gemini AI matches CV vs Job → CVMatchResult (score, skills, analysis)
5. Application saved to Firestore with matchScore
6. Company notified → can view applicants sorted by match score
7. Company shortlists → schedule interview → student notified via FCM

Interview Flow

1. Company schedules interview (Technical/HR/AI Behavioral/Case Study)
2. Student receives notification → opens Interview Screen
3. Gemini generates interview questions based on job requirements
4. Student answers (with proctoring - tab switch detection)
5. Gemini scores answers → InterviewScore with feedback
6. Company sees interview results on Application Details

6. CV Parsing & Skill Matching

CVParsingService.kt (Android - uses Gemini AI)

StepDescription
1. Render PDFPDFBox renders PDF pages to Bitmap images (max 3 pages, 1024px width)
2. Vision AI ParseGemini Vision analyzes images → extracts structured JSON
3. Match CV to JobGemini Text AI compares CV vs job → match score (0-100)
4. FallbackIf AI fails → basic keyword matching algorithm

Scoring criteria:

  • Skills match: 40% weight
  • Experience relevance: 30% weight
  • Education & GPA: 20% weight
  • Projects & certifications: 10% weight

AISkillMatchingService.kt (Android - uses Gemini AI)

  • getSkillMatches() — Matches student skills against all active jobs using AI
  • analyzeStudentSkills() — Comprehensive skill analysis (technical/soft skills, career path)
  • getLearningRecommendations() — Personalized learning suggestions
  • getRealTimeJobSuggestions() — Profile-based real-time job categories
  • Fallback to basic set intersection when AI is unavailable

Feature Details

1. CV Parsing (NLP — no AI needed)

Uses regex + keyword matching to extract:

  • Name: First non-empty line heuristic
  • Email: Regex pattern matching
  • Phone: Pakistani & international number formats
  • Skills: 172 predefined skill keywords matched case-insensitively
  • Education: Section detection + degree keywords
  • Experience: Section detection + year extraction
  • GPA: Pattern matching (X.X/4.0, CGPA: X.X)
  • Certifications: Section + keyword detection

2. CV Parsing — spaCy NER + Regex

  • SpacyNerParser class extends CVParser with spaCy en_core_web_lg NER model
  • Extracts: PERSON → candidate name, ORG → universities/companies, PRODUCT → tech skills
  • Hybrid approach: NER + 172 keyword set for maximum coverage
  • Automatic fallback to regex-based CVParser when spaCy not installed
  • Check: parser.is_using_spacy property to verify NER is active

3. Semantic Matching — BERT Embeddings + Cosine Similarity

  • Primary: BERT sentence-transformers (all-MiniLM-L6-v2) → 384-dim dense embeddings → cosine similarity
  • Fallback: Pure Python character n-gram (bigram + trigram) cosine similarity — zero external deps
  • Model loaded eagerly by default; lazy_load=True for deferred loading
  • Check: matcher.using_bert property to verify BERT is active
  • Composite score: 60% semantic + 40% keyword overlap + experience/education bonus

4. Shortlisting

3. Shortlisting

  • Filters by match score threshold (per-job or global)
  • Optional GPAA minimum, experience minimum
  • Top-K ranking
  • Returns shortlisted vs rejected lists with reasons

4. Job Recommendations

  • Hybrid scoring: 40% skill overlap + 40% semantic similarity + 20% education bonus
  • CV-based or Student-profile-based recommendations
  • Re-ranking on profile changes
  • Filters: location, salary range

Composite Score Formula

# 60% Semantic + 40% Keyword → then 60% of that + 40% structured
base = (cosine_similarity * 60.0) + (skill_overlap_score * 0.4)
total = base * 0.6 + (experience_score + education_score + extra) * 0.4
total = min(total, 100.0)
